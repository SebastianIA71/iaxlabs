Cómo publicar en IAxLabs

Guía operativa — para cuando vuelves después de 3 semanas y no recuerdas nada

uso interno · no indexado
1

Define el slug del nuevo topic

El slug identifica todo el topic y sus labs

El slug sigue siempre el mismo patrón:

0006número topic
DBcategoría (2 letras)
01número lab
vectorsnombre corto
Ejemplo completo: 0006-DB-01-vectors
Categorías usadas hasta ahora: PH (física), DA (datos), CS (computer science), AI (IA), CL (cloud), DB (databases)

Tipos de entrada en el topic:

article

sin slug, solo url a Substack

lab

con slug, carga index.html

qa

con slug, igual que lab

2

Crea las carpetas en /public/experiments/

Una carpeta por cada lab y Q&A del topic

public/
experiments/
0006-DB-01-vectors/
index.html ← el lab
icon.JPG ← imagen del topic (solo en el primer lab)
0006-DB-02-semantic/
index.html
0006-DB-03-qa/
index.html
El icon.JPG solo va en la carpeta del primer lab del topic (order: 1). Es la imagen que aparece en la home de IAxLabs. Se genera en Freepik y se guarda como JPG. Dimensiones recomendadas: 800×600px aprox.
3

Crea el index.html de cada lab

Copia uno existente como base y modifica el contenido

Copia siempre un index.html existente del mismo estilo visual que quieres (light/dark, serif/sans). Los labs de cloud costs usan light mode con serif para títulos. Úsalos como referencia para topics nuevos.

Lo que necesitas cambiar en cada index.html:

  • Título de la página (<title> y <h1>)
  • Subtítulo y descripción del lab
  • El contenido interactivo (simulaciones, ejercicios, Q&A)
  • Los colores del acumulador / sesión si aplica
Convención UI: Lab 1 → light mode, acumulador arriba. Labs 2+ → Georgia serif para títulos, opciones de respuesta en color neutro (no revelar correcto antes de seleccionar).
4

Registra el topic en /data/labs.ts

Sin esto, el topic no aparece en ningún sitio

Añade al final del array en labs.ts el bloque completo del nuevo topic:

// =========================
// 🗄️ TOPIC 6: BASES DE DATOS VECTORIALES
// =========================
{
  name: "Título del artículo en Substack",
  subtitle: "Subtítulo del artículo",
  url: "https://iafont.com/p/slug-del-articulo", // URL real del post
  icons: ["ai", "data"],
  image: "/experiments/0006-DB-01-vectors/icon.JPG",
  type: "article",
  topic: "vectors",
  order: 0,
  topicTitle: "BVD → Búsqueda Semántica",
  topicDescription: "De vectores a memoria inteligente",
},
{
  name: "Nombre del Lab 1",
  subtitle: "Subtítulo corto",
  slug: "0006-DB-01-vectors",
  icons: ["ai", "data"],
  image: "/experiments/0006-DB-01-vectors/icon.JPG",
  type: "lab",
  topic: "vectors",
  order: 1,
},
{
  name: "Nombre del Lab 2",
  subtitle: "Subtítulo corto",
  slug: "0006-DB-02-semantic",
  icons: ["ai", "data"],
  type: "lab",
  topic: "vectors",
  order: 2,
},
{
  name: "Q&A Bases de Datos Vectoriales",
  subtitle: "Lo que nadie explica bien sobre BVD",
  slug: "0006-DB-03-qa",
  icons: ["ai", "data"],
  type: "qa",
  topic: "vectors",
  order: 3,
},
Importante: el campo url del article se rellena con la URL real del post de Substack. Hasta que el artículo esté publicado, déjalo como https://iafont.com/ (raíz). El 1 de mayo se actualizan todos.
5

Haz push y verifica en Vercel

Vercel despliega automáticamente desde main

# En local, desde la raíz del proyecto
git add .
git commit -m "feat: add topic 0006 BVD vectors"
git push origin main
  • Vercel detecta el push y despliega automáticamente (~1-2 min)
  • Verifica en iaxlabs.com que el nuevo topic aparece en la home
  • Verifica que cada lab carga correctamente en iaxlabs.com/labs/0006-DB-01-vectors
  • Verifica el icon.JPG en la card del topic
✓

Checklist de publicación completa

Antes de dar por publicado un topic

  • Artículo publicado en IAfont (Substack)
  • URL real del artículo actualizada en labs.ts
  • Carpetas creadas en /public/experiments/
  • icon.JPG en la carpeta del primer lab
  • Todos los index.html funcionan correctamente
  • Labs verificados en móvil
  • Push a main y deploy verificado en Vercel
★

Destacar un topic a doble página

El topic featured aparece arriba, a ancho completo, antes del grid

En /data/labs.ts, añade featured: true al article del topic que quieres destacar. Quita el del anterior si lo había.

{
  name: "Título del artículo",
  type: "article",
  topic: "vectors",
  featured: true,  // ← solo esto
  // ... resto de campos igual
}
Solo un topic featured a la vez. Push y listo.
